## 2. 风格方向与设计策略

- DESIGN.md 对齐结论：轻量科技感、白底蓝系、强调结构清晰和工程化复用。
- 风格方向：平台母版，不走纯运营风，也不走传统重后台风。
- 页面气质关键词：克制、清楚、稳定、可派生。
- 品牌 / 调性说明：`Omni Platform` 是平台底座，不预设具体行业皮肤。
- 主操作与危险操作层级：主操作固定蓝色，危险操作固定红色，禁止同层抢焦点。
- 信息层级原则：先结论，再入口，再细节；先结构，再数据，再装饰。

## 3. 设计 Token 与组件策略

- 主色 / 辅色方向：主色 `#2563eb`，辅强调 `#7c3aed`，成功 `#16a34a`，危险 `#dc2626`。
- 字体 / 字重方向：中文优先 `IBM Plex Sans SC` 风格，标题 `600/700`，正文 `400/500`。
- 圆角 / 阴影方向：中圆角、轻阴影、强边界。
- 列表 / 卡片 / 表单组件策略：以信息卡片和配置表单为主，不堆砌复杂图表。

## 5. 关键交互

| 场景 | 触发动作 | 系统反馈 | 成功状态 | 失败状态 |
|---|---|---|---|---|
| 查看平台元信息 | 打开接口联调页或请求 `/meta` | 展示项目名、端口、版本 | 信息卡渲染成功 | 展示接口失败与重试入口 |
| 查看服务健康 | 请求 `/health` | 显示服务状态标签 | 显示 `ok` 和时间 | 显示失败原因 |
| 跳转文档中心 | 点击 docs 入口 | 打开 `docs/INDEX.md` 对应页面 | 快速定位阶段文档 | 提示目标文档缺失 |
| 切换端别视图 | 点击 `front/mobile/server` 标签 | 更新当前说明区 | 当前端别高亮且内容切换 | 保持当前页并提示未实现 |

## 6. 状态与异常

| 状态类型 | 展示规则 | 用户可执行动作 |
|---|---|---|
| 加载中 | 骨架屏或轻量 loading，不遮住整体布局 | 等待或取消 |
| 空数据 | 显示“母版待派生”提示和推荐动作 | 去创建首个模块 |
| 无权限 | 明确写无权限原因和申请路径 | 返回首页或申请权限 |
| 接口失败 | 卡片内联错误 + 重试按钮 | 重试、查看日志 |

## 7. 动效与反馈

- 页面切换：淡入和轻位移即可，时长控制在 `180-240ms`
- 弹层 / 抽屉：优先抽屉承载配置编辑，弹层只做确认
- 提交 / 保存 / 删除：提交后按钮进入 loading，结束后用 toast + 卡片状态双反馈
- 长任务 / 异步生成：展示阶段状态、预计耗时和最近更新时间

## 8. 多端适配

| 端 | 布局策略 | 组件差异 | 手势/交互差异 |
|---|---|---|---|
| Web | 双列或三段布局 | 支持侧栏、表格、联调面板 | 鼠标悬停与快捷操作更丰富 |
| APP | 单列优先 | 卡片更紧凑，减少大表格 | 底部操作区固定，支持手势返回 |
| 小程序 | 单列、分步操作 | 减少复杂输入与长表单 | 点击反馈更强，避免 hover 依赖 |
| H5 | 单列 + 吸顶操作条 | 保留核心查看和提交能力 | 强调触控热区与滚动连续性 |

## 9. 可访问性与响应式要求

- 断点策略：`<768px` 单列，`768-1199px` 双列，`>=1200px` 宽屏布局
- 触控热区要求：按钮、标签、列表项最小点击高度 `44px`
- 键盘 / 焦点 / 错误提示要求：可聚焦元素必须有可见焦点态，错误提示贴近字段出现
- 长文本 / 文案溢出处理：标题最多两行，表格说明超出后省略并提供 tooltip

## 10. UI验收标准

- 视觉还原度：符合 `DESIGN.md` 规定的颜色、边界、圆角和字体层级。
- 交互一致性：Web / H5 / Mobile 的关键状态反馈语义一致。
- 文案与提示一致性：健康、错误、空态、待派生提示使用统一术语。
