## 1. Visual Theme & Atmosphere

- 核心气质：平台母版、工程化、可派生、清晰分层。
- 视觉关键词：平台底座、结构清楚、理性、稳定、轻量科技感。
- 情绪目标：让首次接手者第一眼就知道“这是一个可复用的平台母版”。
- 页面密度：中等偏低，优先可读性。
- 用户第一眼应该感知到什么：三端分离、根级编排、文档和资产分层。
- 明确避免什么观感：通用 admin 套壳感、过度营销感、无层次的占位页。

## 3. Typography Rules

- 标题字体：`"IBM Plex Sans SC", "PingFang SC", "Microsoft YaHei", sans-serif`
- 正文字体：`"IBM Plex Sans SC", "PingFang SC", "Microsoft YaHei", sans-serif`
- 数字 / 结果码字体：`"JetBrains Mono", "SFMono-Regular", monospace`
- 字重策略：标题 `600/700`，正文 `400/500`，数据标签 `500/600`
- 标题字距策略：标题略收紧，控制在 `-0.02em` 到 `-0.04em`
- 正文行高策略：正文 `1.6`，卡片说明 `1.5`，表格说明 `1.4`

| 层级 | 用途 | 字号建议 | 字重 | 备注 |
|---|---|---|---|---|
| `display` | 头图 / 主视觉标题 | `40-48px` | `700` | 只用于首页或启动页 |
| `h1` | 页面主标题 | `28-32px` | `700` | 与项目定位、页面主任务绑定 |
| `h2` | 区块标题 | `20-24px` | `600` | 用于模块分区 |
| `h3` | 卡片标题 | `16-18px` | `600` | 用于功能卡片、面板标题 |
| `body` | 正文 | `14-16px` | `400` | 默认阅读层级 |
| `meta` | 标签 / 辅助说明 | `12-13px` | `500` | 用于状态、说明、二级标签 |

## 4. Component Stylings

### Buttons

- Primary Button：实底 `color-primary`，白字，圆角 `12px`，hover 切到 `color-primary-strong`
- Secondary Button：白底蓝边，文字使用 `color-primary`，用于次级提交或查看详情
- Ghost Button：透明底，仅保留文字与轻边框，用于表格工具条和低优先级操作
- 禁止：渐变荧光按钮、过厚投影、危险操作与主按钮同色

### Cards / Panels

- 主卡片：白底、细边框、轻阴影，顶部可带标题与操作区
- 信息卡：左对齐排版，优先展示结论、状态、下一步动作
- 数据卡：允许使用浅色背景强调数字，但不做重仪表盘风格
- 禁止：无边界大白板、过度玻璃拟态、纯占位卡片

### Inputs / Forms

- 输入框风格：白底、`1px` 描边、`10-12px` 圆角，内边距中等
- 焦点态：蓝色描边 + 极浅蓝光晕，不能跳动
- 错误态：红色描边 + 错误文案靠近字段，不只靠 toast
- 禁止：高饱和填充输入框、过细点击区域、隐藏标签

### Navigation / Chips / Tags

- 顶部导航：强调项目名、当前端别、环境与快速入口
- 标签 / Chip：使用低饱和背景色区分 `front / mobile / server / docs`
- 分段区块：推荐用左侧色条或小标题，不依赖重分割线
- 禁止：同时出现多层 tabs、胶囊满屏堆叠、彩虹标签

## 5. Layout Principles

- 容器宽度策略：桌面端主内容 `1200-1280px`，表单与说明页优先窄容器
- 留白策略：页面外边距宽于组件内边距，保持平台感和结构感
- 栅格策略：桌面端 12 栅格，移动端单列优先，卡片最多双列
- 单屏信息量上限：首屏聚焦一个主任务加两个辅助手段
- 首屏重点：项目定位、三端入口、部署状态、文档入口
- 组件之间的节奏：大区块 `32px`，区块内 `20-24px`，字段组 `12-16px`

## 6. Depth & Elevation

- 卡片圆角策略：主卡片 `16px`，按钮 `12px`，输入框 `10-12px`
- 阴影策略：只保留轻阴影，重点靠边界和层级，不靠大面积阴影
- 模糊 / 镜面 / 渐变策略：可在启动页或架构封面轻量使用，业务页默认禁用
- 哪些地方可以重、哪些地方必须轻：启动页头图可略重，后台列表和表单必须轻

## 7. Do's and Don'ts

### Do

- 明确展示平台结构、文档入口和部署关系
- 用统一 token 保持 front、mobile、server 的视觉连续性
- 在关键页面优先展示结论、状态和下一步动作

### Don't

- 不做通用 admin 模板皮肤化复制
- 不把首页做成纯指标大盘而失去平台母版定位
- 不使用高噪点背景、重紫色主视觉或无意义动画

## 8. Responsive Behavior

- 移动端主断点：`<768px`
- 桌面端主断点：`>=1200px`
- 标题缩放规则：`display/h1` 在移动端下调一档，保持两行内可读
- 按钮换行规则：主操作优先独占一行，辅助操作次行平铺
- 卡片折叠规则：多列卡片收为单列，摘要信息先于次级说明
- 触控热区要求：可点击区域不低于 `44px`

## 9. Agent Prompt Guide

- 一句话风格提示：做成“可直接派生的平台母版”，而不是“泛化后台模板”。
- 生成页面时必须遵守：优先保留三端分层、文档入口、部署入口和稳定的工程化观感。
- 允许自由发挥的范围：首页封面、架构展示区、轻动效和说明卡片排版。
- 如果视觉与内容冲突，优先级规则：信息结构优先于装饰，功能识别优先于视觉炫技。
