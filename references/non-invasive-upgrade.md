# Non-Invasive Existing Project Upgrade Rules

## Core Rule

Treat existing projects as production-adjacent assets. Add AI handoff capability without changing the project identity or source shape.

## Allowed by Default

- Add `AGENTS.md`
- Add `CLAUDE.md`
- Add `START-HERE.md`
- Add missing `assets/` folders
- Add missing `docs/ai-upgrade/`
- Add an upgrade report
- Add architecture/design/process images generated by `image_gen`
- Add or refresh `graphify-out/GRAPH_REPORT.md` when useful
- Enhance README while preserving existing content

## Not Allowed Without Explicit User Approval

- Move source directories
- Rename business modules
- Replace framework or build tooling
- Rewrite deployment strategy
- Delete existing docs/scripts/config
- Format the whole repository
- Rewrite README from scratch when one already exists
- Change package names, group IDs, module names, namespaces, or database names

## Existing README Strategy

- If missing: create a complete README.
- If outdated: append or patch missing sections.
- If poor but still valuable: preserve content and add a new structured section.
- If full rewrite is necessary: ask or create a backup first.

## Upgrade Report

Always produce a report under:

```text
docs/ai-upgrade/report-老项目AI能力升级.md
```

The report must list:

- scanned scope
- files added
- files changed
- files intentionally not changed
- risks
- next recommendations
